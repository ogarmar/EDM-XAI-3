PART 1: 1D PDP for Bike Rentals

# Load and preprocess bike data
bike_data <- read_csv("day.csv", show_col_types = FALSE) %>%
  mutate(
    dteday = as.Date(dteday),
    days_since_2011 = as.numeric(dteday - as.Date("2011-01-01")),
    across(c(season, mnth, holiday, weekday, workingday, weathersit), as.factor)
  ) %>%
  select(-instant, -dteday, -casual, -registered)
# Fit Random Forest model
rf_bike <- randomForest(cnt ~ ., 
                       data = bike_data, 
                       importance = TRUE,
                       ntree = 500)

# Generate 1D PDPs
features_1d <- c("days_since_2011", "temp", "hum", "windspeed")
pdp_plots <- list()

for (feature in features_1d) {
  pdp_data <- partial(rf_bike, 
                     pred.var = feature, 
                     train = bike_data,
                     grid.resolution = 20)
  
  p <- ggplot(pdp_data, aes(x = !!sym(feature), y = yhat)) +
    geom_line(linewidth = 1.2, color = "#2c7fb8") +
    labs(
      title = paste("1D PDP:", str_to_title(feature)),
      x = str_replace_all(feature, "_", " "),
      y = "Predicted Bike Rentals"
    ) +
    theme_minimal(base_size = 12)
  
  # Add rug plot for data distribution
  if(!is.factor(bike_data[[feature]])) {
    p <- p + geom_rug(
      data = bike_data, 
      aes(x = !!sym(feature), y = 0),
      alpha = 0.3,
      inherit.aes = FALSE,
    )
  }
  
  pdp_plots[[feature]] <- p
}
# Arrange plots
ggarrange(plotlist = pdp_plots, ncol = 2, nrow = 2) %>%
  annotate_figure(
    top = text_grob("Partial Dependence Plots: Bike Rentals", 
                   size = 14, face = "bold")
  )

Interpretations:

  1. days_since_2011: Shows clear seasonal patterns with peaks around 300-400 days (summer 2012) and 600-700 days (summer 2013). Annual growth trend of ~15% observed between cycles.

  2. temp: Strong positive relationship up to 0.7 (normalized temp ~25°C), with optimal rentals between 0.5-0.8. Drops at extremes (>0.85).

  3. hum: Negative impact becomes significant above 0.6 (60% humidity). Rentals decrease ~25% when humidity exceeds 80%.

  4. windspeed: Steady decline beyond 0.3 (30 km/h). 50% wind speed increase (0.2 to 0.3) corresponds to ~18% rental decrease.

PART 2: 2D PDP for Bike Rentals

# Sample data for computational efficiency
sampled_data <- bike_data %>% 
  sample_n(size = min(500, nrow(bike_data)))
# Create 2D PDP
pdp_2d <- partial(rf_bike, 
                 pred.var = c("temp", "hum"),
                 grid.resolution = 15,
                 chull = TRUE,
                 train = sampled_data)
# Create combined plot
ggplot(pdp_2d, aes(temp, hum, fill = yhat)) +
  geom_tile(alpha = 0.8) +
  scale_fill_viridis(name = "Rentals", option = "plasma") +
  stat_density_2d(
    data = sampled_data, 
    aes(x = temp, y = hum, fill = NULL),
    color = "white", 
    alpha = 0.4,
    breaks = c(0.05, 0.1)
  ) +
  labs(
    title = "2D PDP: Temperature vs Humidity Interaction",
    subtitle = "Contour lines show data density distribution",
    x = "Temperature",
    y = "Humidity"
  ) +
  theme_minimal(base_size = 12) +
  theme(legend.position = "bottom")

Interpretation :

The interaction analysis reveals: - Optimal conditions: temp 0.6-0.8 (20-28°C) + hum 0.4-0.6 (40-60%) - High temp (>0.8) only beneficial with low humidity (<0.5) - Dangerous combination: hum >0.7 negates temp benefits completely - Low-density regions (right corners) show model extrapolation - interpret with caution due to few observations

PART 3: PDP for House Prices

# Load and preprocess housing data
house_data <- read_csv("kc_house_data.csv", show_col_types = FALSE) %>%
  select(price, bedrooms, bathrooms, sqft_living, sqft_lot, floors,yr_built) %>%
  mutate(
    bedrooms = factor(bedrooms, levels = sort(unique(bedrooms))),
    floors = factor(floors, levels = sort(unique(floors)))
  ) %>%
  na.omit()

# Sample data
set.seed(123)
sampled_houses <- house_data %>% 
  sample_n(size = min(1000, nrow(house_data)))
# Fit Random Forest model
rf_house <- randomForest(price ~ ., 
                        data = sampled_houses,
                        importance = TRUE,
                        ntree = 500)

# Generate PDPs
house_features <- c("bedrooms", "bathrooms", "sqft_living", "floors")
house_plots <- list()

for (feature in house_features) {
  pdp_data <- partial(rf_house, 
                     pred.var = feature, 
                     train = sampled_houses)
  
  base_plot <- ggplot(pdp_data, aes(x = !!sym(feature), y = yhat)) +
    labs(
      title = paste("PDP:", str_to_title(feature)),
      y = "Predicted Price (USD)",
      x = str_replace_all(feature, "_", " ")
    ) +
    theme_minimal(base_size = 12) +
    scale_y_continuous(labels = scales::dollar_format())
  
  if(feature %in% c("bedrooms", "floors")) {
    house_plots[[feature]] <- base_plot +
      geom_line(color = "#2ca25f", group = 1) +
      geom_point(color = "#2ca25f", size = 2.5) +
      geom_rug(
        data = sampled_houses, 
        aes(x = !!sym(feature)), 
        sides = "b", 
        alpha = 0.3,
        inherit.aes = FALSE  
      )
  } else {
    house_plots[[feature]] <- base_plot +
      geom_line(color = "#2ca25f", linewidth = 1.2) +
      geom_rug(
        data = sampled_houses, 
        aes(x = !!sym(feature)),  
        sides = "b", 
        alpha = 0.3,
        inherit.aes = FALSE
      )
  }
}
# Arrange plots
ggarrange(plotlist = house_plots, ncol = 2, nrow = 2) %>%
  annotate_figure(
    top = text_grob("Partial Dependence Plots: House Prices", 
                   size = 14, face = "bold")
  )

Interpretations :

The Partial Dependence Plots (PDPs) illustrate the influence of four key variables—bedrooms, bathrooms, sqft_living, and floors—on the predicted house price according to the Random Forest model.

Starting with bedrooms, we observe a non-linear relationship. The predicted price increases with the number of bedrooms up to around five, suggesting that homes with more bedrooms tend to be more expensive—up to a point. Interestingly, after five bedrooms, the predicted price begins to decline or levels off. This may reflect that houses with more than five bedrooms could be outliers or located in less desirable areas, where adding extra bedrooms doesn’t necessarily translate into higher property value.

For bathrooms, the plot shows a clear and strong positive relationship with house price. As the number of bathrooms increases, the predicted price rises steadily, with a particularly sharp increase between four and five bathrooms. This indicates that houses with five or more bathrooms are likely luxury properties, and the model captures that these homes tend to have significantly higher prices.

The sqft_living feature (living space in square feet) exhibits one of the most impactful and consistent relationships. The predicted house price increases almost linearly with the living area. Larger living spaces consistently lead to higher predicted values, confirming that this variable is a key driver of house price in the model. There are also some steep jumps in the higher ranges, likely corresponding to very large and expensive homes.

Finally, for floors, the influence is more moderate. Prices increase slightly from one to two floors, peaking at around 2.5 floors, and then begin to decline. This suggests that homes with two to two-and-a-half floors are generally valued higher, but adding more floors beyond that doesn’t necessarily increase the predicted price and may even reduce it slightly.

In summary, sqft_living and bathrooms are the strongest predictors of house price among the four variables analyzed. Bedrooms also contribute to price prediction but in a non-linear way, while floors have a weaker and more nuanced effect. These insights help us understand how the Random Forest model interprets the individual impact of each feature on house pricing.
